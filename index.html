    Introduction

    The Xenithpay API is organized around REST. Our API has predictable, resource-oriented URLs, and uses HTTP response codes to indicate API errors. We use built-in HTTP features and HTTP verbs, which are understood by off-the-shelf HTTP clients. JSON is returned by all API responses, including errors.

    Feel free to reach out to us at help@xenithpay.com if you have any inquiries.

    Build Signature

    View the example on the right for reference.

    How to Generate Signature on Postman Pre-Script

    secretKey =
  "sk-760a8a0c8826b2e9dd8fe3ed0c66976f0c3d4a0443056674b7c132a25ddbbaf6eed1a410e02609a0a9791d7683c5de83860574bf6a92ddf8ea5a14360a96ac3c";
// Get the current timestamp in RFC3339 format
var timestamp = new Date().toISOString();
// Get the request method and URI
var method = pm.request.method;
var uri = pm.request.url.getPathWithQuery();
// Concatenate the method, URI, and timestamp
var signaturePayload = method + "\n" + uri + "\n" + timestamp;
// Create a HMAC SHA256 hash of the signature payload
var hash = CryptoJS.HmacSHA256(signaturePayload, secretKey);
// Encode the hash in base64
var signature = CryptoJS.enc.Base64.stringify(hash);
// Set the timestamp and signature as environment variables
pm.environment.set("request_timestamp", timestamp);
pm.environment.set("request_signature", signature);

// use the environment on your request.

    Format

    Format Example
    HTTP METHOD + "\n" + URI + "\n" + TIMESTAMP POST
    /v1/payins
    2024-05-10T16:42:22.802Z

    Sign the Signature

    signaturePayload= "POST\n/v1/payins\n2024-05-10T16:42:22.802Z"
    var hash = CryptoJS.HmacSHA256(signaturePayload, <Secret Key>)
    signature = CryptoJS.enc.Base64.stringify(hash);

    Now finally, add the signature to every request header when calling the API for Zenith

    Xenith-Request-Signature = signature

    Status Code Error Code Meaning
    400 API_VALIDATION_ERROR Invalid request errors arise when your request has invalid parameters.
    400 INVALID_AMOUNT The amount entered is not supported.
    401 INVALID_API_KEY The API key provided is incorrect.
    401 INVALID_SIGNATURE The request signature is invalid.
    401 UNKNOWN_IP_ADDRESS The IP address is not on the allowed list.
    401 UNAUTHORIZED_ACCESS Access is denied.
    403 FORBIDDEN Insufficient permissions to proceed with the action.
    404 DATA_NOT_FOUND The requested data/item does not exist.
    405 NOT_ALLOWED The endpoint was accessed with an incorrect HTTP method.
    408 REQUEST_TIMEOUT The request is taking longer than expected. Please contact support or try again.
    409 DATA_DUPLICATE/DATA_ALREADY_EXISTS The data is already exist.
    422 INSUFFICIENT_BALANCE Insufficient balance to proceed with the action.
    500 SERVER_ERROR An issue occurred on our server. Please contact support or try again later.
    501 NOT_IMPLEMENTED The service requested is not supported yet.
    503 SERVICE_UNAVAILABLE The service is temporarily offline for maintenance. Please contact support or try again later.
