Webhooks

Xenith uses webhooks to notify your application whenever an event occurs on your account. You can set up webhooks for events that Xenith doesnâ€™t automatically notify you about, such as when a Pay In is successful, expired, or failed, and when a Pay Out is successful or failed.

Webhooks are HTTP callbacks sent to an endpoint on your server. Xenith requires you to use HTTPS endpoints with TLSv1.2 or TLSv1.3 for secure communication.

Check Webhook Signature

Xenith signs all webhook events by including a signature in the request headers. This allows you to verify that the events were sent by Xenith. Follow these steps to verify the signature:

  1. Extract the x-signature from the request header.
  2. Compute an HMAC using the SHA-256 hash function with the x-signature and the raw JSON payload.
  3. Compare the x-signature from the header with your computed signature. If they match, check the difference between the current timestamp and the received timestamp to ensure it falls within your allowed tolerance.

When generating the expected signature, ensure you're using the raw, unmodified JSON payload.

Note: Many libraries format the JSON when parsing it, so it's best to verify the signature before any transformation occurs.

Delivery Attempts and Retries

Xenith attempts to deliver your webhook eleven times with exponential backoff between each interval. We will stop retrying after receiving a response from your server or if there is still no response after the final attempt.
